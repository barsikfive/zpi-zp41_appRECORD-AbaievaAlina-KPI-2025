Власний WEB-застосунок

Тема

Веб-додаток сервісу цифрової дистрибуції відеоігор.

Опис бізнес-логіки:

  • Головна сторінка містить каталог цифрових товарів для дистрибуції.
  • Розділ FEATURED анімований та стилізований під актуальні події, фестивалі або акції.
  • Розділ ON SALE стилізований для виведення інформації про топ товарів з великою знижкою (від 50%).
  • Розділ TOP SELLERS пропонує користувачу найгарячіші позиції.
  • Меню каталога має бути пов'язане з іншими сторінками сервісу.
  • Користувачу пропонується оформлення підписки на особливі пропозиції та акції.

Вигляд

Теорія

<table> визначає таблицю.
<th> визначає клітинку заголовка в таблиці
<tr> визначає рядок у таблиці
<td> визначає клітинку в таблиці
<caption> визначає заголовок таблиці
<thead> групує вміст заголовка в таблиці
<tbody> групує основний вміст у таблиці
<tfoot> групує вміст нижнього колонтитула в таблиці
colspan та rowspan визначають охоплення стовпців та рядків однією клітинкою таблиці

Вигляд

Теорія

Зображення має обов'язковий атрибут src, який містить шлях до зображення.
Атрибут alt містить текстовий опис зображення, який не обов'язковий, але неймовірно корисний для доступності - програми читання з екрану читають цей опис своїм користувачам, так вони знають яке зображення показано, і так само воно відображається на сторінці, якщо зображення не може бути завантажене за якою або причини.

Теорія

Форма HTML використовується для збору даних, введених користувачем. Введені користувачем дані найчастіше надсилаються на сервер для обробки.
label є підписом для поля введення даних, в той час як input відповідає за саме поле та його тип.

Висновки

HTML-документ

HTML-документ складається з кількох основних елементів, таких як html, head, та body. Вони визначають загальну структуру веб-сторінки. Внутрішній вміст документа оформлюється за допомогою різноманітних тегів, що дозволяють включати заголовки, абзаци, зображення, таблиці, списки, посилання та інші елементи.

Посилання, таблиці, зображення

Робота з посиланнями в HTML дозволяє створювати інтерактивні елементи, які ведуть користувача до інших сторінок або зовнішніх ресурсів. Для цього використовуються теги <a>, що надають можливість створювати гіперпосилання. Крім того, HTML дозволяє вставляти зображення за допомогою тега <img>, що дає змогу візуалізувати інформацію, що важлива для звіту чи презентації. Таблиці, оформлені за допомогою тегів <table>, <tr>, <td>, надають можливість структуровано подати дані, що є зручним для відображення різних результатів або статистичних показників.

Списки

HTML також підтримує використання списків, як нумерованих, так і ненумерованих, за допомогою тегів <ol> та <ul>, що дають змогу впорядкувати інформацію. Списки часто використовуються для відображення етапів виконання задач чи переліків важливих елементів, що підвищує читабельність документа. Після оволодіння базовими елементами HTML, можна створювати більш складні документи, де поєднуються таблиці, списки, зображення і посилання для ефективної презентації результатів лабораторних робіт та інших проектів.

Форми

Форми в HTML використовуються для збору та обробки даних, які користувачі вводять на веб-сторінках. Вони створюються за допомогою тегу <form>, всередині якого можна додавати різні елементи введення, такі як текстові поля (<input>), випадаючі списки (<select>), радіо-кнопки (<input type="radio">) та чекбокси (<input type="checkbox">). Атрибути тегів, такі як action і method, визначають місце та метод передачі даних. Форми використовуються для інтерактивності на веб-сторінках, наприклад, для збору відгуків, реєстрацій чи фільтрації інформації на основі введених параметрів.

Зовнішня таблиця стилів

Зовнішні стилі визначені в елементі <link> всередині розділу <head> HTML сторінки. Файл не повинен містити жодних HTML-тегів. Файл таблиці стилів має бути збережено з розширенням .css.

Внутрішня таблиця стилів

Внутрішня таблиця стилів може використовуватися, якщо одна окрема HTML сторінка має свій унікальний стиль і цей стиль більше ніде не використовується на інших сторінках.

Вбудовані стилі

Вбудований стиль може використовуватися для застосування унікального стилю для окремого елементу на HTML сторінці. Щоб використовувати вбудовані стилі, додайте атрибут стилю до відповідного елементу (тегу). Атрибут style може містити будь-яку властивість CSS.

Порядок використання каскадних стилів

Вбудований стиль має найвищий пріоритет та перевизначає зовнішні та внутрішні стилі й налаштування браузера за замовчуванням.

Селектор - це частина CSS-правила, яка повідомляє браузеру, до якого елементу веб-сторінки буде застосований CSS стиль. Тобто, селектор - це вибірка та формальний опис того елемента чи групи елементів, до яких будуть застосовані CSS стилі.

Засвоєння основ роботи з CSS-селекторами

У цій лабораторній роботі було вивчено основи роботи з різними видами CSS-селекторів: селектори тегів, класів, ідентифікаторів, а також сусідні, дочірні, атрибутні й універсальні селектори. Це дозволило зрозуміти, як точно визначати стилі для елементів HTML-документа, покращуючи їхній зовнішній вигляд і адаптивність.

Підключення стилів і їхній вплив

Було розглянуто три основні способи підключення стилів: зовнішні таблиці стилів, внутрішні стилі та вбудовані стилі. Кожен з них має свої переваги та обмеження, і їх вибір залежить від вимог до структури проєкту. Було також засвоєно, як каскадність і пріоритетність стилів впливають на кінцевий вигляд елементів сторінки.

Оформлення текстових елементів і роботи зі шрифтами

В процесі роботи вивчено підключення локальних шрифтів за допомогою правила @font-face та використання шрифтів з Google Fonts. Це допомогло набути навичок налаштування тексту, включаючи зміну його кольору, розміру, інтервалів, а також шрифтів, що створює унікальний стиль для веб-додатків.

Дизайн списків, таблиць і фону

Було вивчено стилізацію таблиць, списків та елементів фону. Завдяки цьому вдалося додати до веб-сторінки більше візуальної структури та підвищити її зручність для користувачів. Використання CSS дозволило зробити ці елементи адаптивними та естетично привабливими.

Просунутий CSS: тіні, анімація, переходи

Досліджено просунуті можливості CSS, такі як створення тіней, анімацій, плавних переходів і стилізації інтерактивних елементів. Це дало змогу покращити взаємодію користувача зі сторінкою та додати професійний вигляд веб-додатку.

Верстка за допомогою таблиць

Переваги:

  • Легкість створення симетричних структур: Таблиці добре підходять для створення сіток, де елементи повинні бути чітко вирівняні, наприклад, таблиці даних чи суворо структуровані макети.
  • Сумісність зі старими браузерами: Таблиці підтримуються всіма браузерами, навіть дуже старими, що робить їх універсальним інструментом.
  • Простота у навчанні: Для початківців таблиці можуть бути легшим підходом через їх передбачувану структуру та відсутність потреби використовувати складні CSS-властивості.

Недоліки:

  • Відсутність гнучкості: Таблиці важко адаптувати до сучасних вимог адаптивного дизайну. Їхні розміри погано підходять для різних екранів (наприклад, смартфонів).
  • Низька продуктивність: Браузери витрачають більше часу на рендеринг таблиць, оскільки їм потрібно обробляти складні вкладення елементів.
  • Семантичні проблеми: Таблиці призначені для представлення табличних даних, а не для верстки інтерфейсів, що може негативно впливати на SEO та доступність.
  • Складність внесення змін: Зміна макета часто вимагає повного переписування структури HTML.

Верстка за допомогою блоків (плаваючих блоків)

Переваги:

  • Гнучкість: Блокова верстка дозволяє створювати адаптивні макети, які легко підлаштовуються під різні розміри екранів.
  • Сучасні підходи: Використання CSS для плаваючих блоків дозволяє створювати складні макети, зберігаючи чистий і зрозумілий HTML.
  • Продуктивність: Блоки легші для браузера в рендерингу, особливо якщо використовуються сучасні технології, такі як Flexbox або Grid.
  • Семантика: Блоки легко вписуються в сучасну практику створення семантично правильного HTML, що позитивно впливає на SEO та доступність.

Недоліки:

  • Складність для початківців: Використання плаваючих блоків вимагає знання CSS та кращого розуміння потоку документа.
  • Можливі проблеми з вирівнюванням: Якщо неправильно використати властивості CSS, блоки можуть перекриватися або неправильно вирівнюватися.
  • Старі браузери: Плаваючі блоки (особливо з використанням Flexbox чи Grid) можуть не підтримуватися застарілими браузерами.
  • Додатковий CSS: Для створення складних макетів потрібно писати більше CSS-коду, ніж у випадку таблиць.

Верстка за технологією FLEXBOX

Переваги:

  • Простота вирівнювання: Flexbox дозволяє легко вирівнювати елементи по вертикалі та горизонталі без використання додаткових хаків або таблиць.
  • Гнучкість: Елементи автоматично підлаштовуються під розмір контейнера, що робить його ідеальним для адаптивного дизайну.
  • Підтримка розподілу простору: Легко розподіляти елементи рівномірно або за певним правилом (наприклад, space-between, space-around).
  • Порядок елементів: Можна змінювати порядок елементів у контейнері, не впливаючи на HTML-структуру.

Недоліки:

Адаптивна верстка

Адаптивна верстка — це підхід до створення веб-сайтів, які автоматично підлаштовуються під різні розміри екранів. Завдяки цьому сайти коректно відображаються як на десктопах, так і на планшетах чи смартфонах. Основна мета — забезпечити комфортний користувацький досвід незалежно від пристрою.

Медіа-запити

Медіа-запити — це CSS-інструмент, який дозволяє змінювати стилі залежно від властивостей пристрою (розмір екрану, орієнтація тощо). Наприклад, за допомогою медіа-запитів можна задавати різні макети для мобільних і десктопних версій сайту. Вони є основою адаптивного дизайну.

Медіа-типи

Медіа-типи визначають, до яких пристроїв застосовуються стилі, наприклад: screen для екранів, print для друку чи speech для синтезу голосу. Це дозволяє застосовувати специфічні стилі для різних типів пристроїв, оптимізуючи їхній вигляд і функціональність.

Медіа-функції

Медіа-функції — це частина медіа-запитів, яка дозволяє задавати конкретні умови, такі як min-width, max-width, orientation, чи aspect-ratio. Вони надають більше гнучкості в адаптації стилів під специфічні характеристики пристрою.

Метатег viewport

Метатег <meta name="viewport" content="width=device-width, initial-scale=1.0"> задає параметри масштабу та розміру сторінки для мобільних пристроїв. Без цього тегу сайт на смартфонах відображатиметься неправильно, оскільки браузери за замовчуванням масштабують сторінки під стандартну ширину в 980px.

Інструменти розробника

Інструменти розробника (наприклад, DevTools у браузерах) допомагають тестувати адаптивність сторінок. Вони дозволяють емулювати різні розміри екранів, аналізувати медіа-запити, змінювати стилі в реальному часі та знаходити помилки у верстці, що значно полегшує процес розробки.

Стратегія Mobile First

Mobile First — це підхід до дизайну, де розробка починається з мобільної версії, а потім поступово додаються стилі для більш широких екранів. Цей підхід враховує зростаюче використання мобільних пристроїв і дозволяє створювати оптимізовані сайти для мобільних користувачів.

function checkBrackets(str) {
    const stack = [];
    
    const matchingBrackets = {
        ')': '(',
        '}': '{',
        ']': '['
    };

    for (let char of str) {
        
        if (char === '(' || char === '{' || char === '[') {
            stack.push(char);
        } 
        
        else if (char === ')' || char === '}' || char === ']') {
            
            if (stack.length === 0 || stack[stack.length - 1] !== matchingBrackets[char]) {
                return false;
            }
            stack.pop();
        }
    }

    return stack.length === 0;
}
